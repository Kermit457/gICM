### 1. Build REST and GraphQL APIs

Design and implement RESTful and GraphQL APIs following industry best practices with proper versioning, documentation, and error handling.

**Best Practices:**
- Use semantic HTTP methods (GET, POST, PUT, PATCH, DELETE) correctly
- Implement API versioning from the start (URL-based: `/api/v1/` or header-based)
- Return proper HTTP status codes (200, 201, 400, 401, 403, 404, 500)
- Use consistent response formats across all endpoints
- Implement request validation with libraries like Zod or Joi
- Add rate limiting to prevent abuse
- Enable CORS with proper configuration
- Use compression middleware (gzip/brotli) for responses
- Implement proper pagination for list endpoints
- Document APIs with OpenAPI/Swagger
